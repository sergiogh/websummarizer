This is particularly significant as quantum chemistry applications increasingly rely on hybrid workflows and as exascale heterogeneous architectures become the norm. **Technical Approach** The technical innovation centers on porting the most computationally intensive component of sample-based quantum diagonalization—the iterative diagonalization of large, configuration-selected Hamiltonians using the Davidson algorithm—to GPU architectures. The authors employ OpenMP 5.0+ target offload directives to maintain portability and minimize code divergence between CPU and GPU paths. Key optimizations include: (1) flattening deeply nested C++ data structures (such as std::vector>) into contiguous, offset-indexed arrays for efficient GPU memory access; (2) implementing a persistent, device-resident cache of electronic configuration bit-strings to eliminate redundant computation; (3) fully device-side evaluation of Hamiltonian matrix elements, including custom implementations of quantum chemistry routines (e.g., Slater-Condon rules) compatible with OpenMP device code; and (4) careful management of data movement to ensure that all large, constant data (integrals, configuration lists, excitation lists) are transferred once and remain resident on the GPU throughout the iterative process. The persistent configuration cache alone provides a 1.52× speedup on CPU, while the GPU implementation delivers an additional 63× speedup relative to the optimized CPU baseline. Strong scaling is demonstrated up to 128 nodes (1024 GPUs) for a 1.52×10⁹ configuration problem, with parallel efficiency of approximately 75%.
